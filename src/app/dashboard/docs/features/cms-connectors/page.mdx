# CMS Connectors

<Subtitle>Push recommendations and schema markup directly to your content management system.</Subtitle>

## Overview

CMS Connectors let you apply Citatra's recommendations and generated schema markup directly to your website without leaving the dashboard. Instead of copying and pasting code between tools, you can push changes to your CMS in a single click.

## Supported Platforms

- **WordPress** — connects via the WordPress REST API

More platforms are coming soon. Each connector requires API credentials from your CMS to authenticate and authorize changes.

## Setting Up a Connector

1. Navigate to **Settings → CMS Connectors** in your workspace
2. Select your CMS platform from the list
3. Enter your site URL and API key (or application password for WordPress)
4. Click **Test Connection** to verify credentials
5. Once connected, you can start pushing content from the Recommendations and Audit pages

## What You Can Push

Connectors support pushing the following types of content:

- **Schema markup** — structured data generated by audits and recommendations
- **Meta descriptions** — optimized descriptions based on AI analysis
- **Structured content updates** — content improvements suggested by recommendations

Each push targets a specific page on your site and modifies only the relevant section.

## Safety & Rollback

All pushes create a revision in your CMS, so you can roll back any change from your CMS's revision history. Citatra never deletes existing content — it only adds or updates the specific elements you approve.

<Info>CMS Connectors are optional. You can always copy generated markup manually and paste it into your CMS.</Info>
